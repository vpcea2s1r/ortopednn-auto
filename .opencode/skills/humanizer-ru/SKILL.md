# Humanizer-ru — очеловечивание текста (v2.3.0)

Скилл для редактирования русскоязычного текста со следами работы ИИ. Цель — сделать текст естественным, не искажая смысла.

## Когда применять
- Текст на русском выглядит механическим, сухим или шаблонным
- Нужно проверить текст, сгенерированный другой нейросетью
- Пользователь просит «очеловечить», «переписать», «убрать следы ИИ»
- Текст готовится к публикации (статья, пост, письмо, документ)
- В тексте видны однозначные маркеры копирования из чат-бота

## Когда не применять
- Текст не на русском языке
- Исходный код, конфигурационные файлы, технические логи
- Юридические документы, нормативные акты, договоры
- Художественная проза, поэзия, литературные эссе

## Дерево решений
```
Получили текст
  ↓
Это русский? — нет → отказ
  ↓ да
Жанр? — код / конфиг → отказ
       — договор / нормативный акт → только #16-21 (стиль/разметка), не трогать #8 канцелярит
       — художка / поэзия → не применять #13 правило трёх, #16 длинное тире
       — публицистика / маркетинг / блог → полный набор
  ↓
Прогнать regex по chatbot-artifacts
  ↓
Найден хоть один однозначный маркер? — да → удалить, проверить остальной текст
  ↓ нет
Сосчитать мягкие признаки по категориям
  ↓
0–2 признака → текст вероятно человеческий, не править
3–5 признаков → выборочно править критические (🔴), оставить остальное
6+ признаков → переписать целиком с сохранением фактов
```

## Главное правило
**Ни один отдельный мягкий признак не даёт достаточного основания для вердикта «текст написан ИИ».**

## Пять ключевых принципов правки
1. **Удалять мусор.** Убирать вводные фразы-пустышки и слова-костыли.
2. **Ломать шаблоны.** Избегать парных сравнений, драматических списков, риторических подводок.
3. **Менять ритм.** Чередовать длину предложений. Два пункта лучше трёх. Разнообразить концовки абзацев.
4. **Доверять читателю.** Констатировать факты прямо. Избегать разжёвывания и оправданий.
5. **Никаких слоганов.** Если фраза звучит как пафосный слоган — переписать.

## 29 паттернов AI-текста

### Контентные (#1–9)
| # | Крит. | Паттерн | Пример |
|---|-------|---------|--------|
| 1 | 🔴 | Усреднение (puffery) | «настоящий титан» → «главный архитектор процессора ARM» |
| 2 | 🟡 | Преувеличение значимости | «стал ключевым моментом в истории» → «компания открылась в 1989 году» |
| 3 | 🟡 | Акцент на известности | «цитировали NYT, BBC, Forbes» → конкретная цитата с источником |
| 4 | 🟡 | Поверхностный анализ (причастные обороты) | «символизируя связь с землёй» → убрать |
| 5 | 🟡 | Рекламный язык | «жемчужина региона» → конкретный факт |
| 6 | 🔴 | Размытые атрибуции | «эксперты считают» → «по данным исследования Института экологии 2019 года» |
| 7 | 🟢 | Шаблонные «вызовы и перспективы» | «несмотря на трудности, продолжает процветать» → конкретика |
| 8 | 🟡 | Канцелярит | «осуществляет деятельность» → «работает» |
| 9 | 🟡 | Текст о тексте | «список 10 лучших фильмов — это кураторская подборка…» → сразу к делу |

### Языковые (#10–15)
| # | Крит. | Паттерн | Пример |
|---|-------|---------|--------|
| 10 | 🔴 | Злоупотребление машинной лексикой | «безусловно, в контексте… крайне важно раскрыть потенциал» → просто |
| 11 | 🟡 | Избегание «есть/это» | «выступает в качестве» → «это» |
| 12 | 🟡 | Отрицательные параллелизмы | «не просто инструмент, а революция» → убрать |
| 13 | 🔴 | Правило трёх | «доклады, дискуссии и нетворкинг» → «доклады и дискуссии» |
| 14 | 🟢 | Погоня за синонимами | «герой… протагонист… центральный персонаж» → одно слово |
| 15 | 🟡 | Меризм (ложные диапазоны) | «от сингулярности до тёмной материи» → упростить |
| 15a | 🟡 | Нелогичные деепричастия | «используя этот метод, результаты улучшаются» → «когда команда применила метод» |
| 15b | 🟡 | Каскад смягчений | «возможно, в некоторых случаях, это может быть…» → «в исследовании 2023 года это повысило…» |
| 15c | 🟡 | Связки-переходы | «однако стоит отметить…», «в заключение хочется…» → убрать |
| 15d | 🟢 | Резкая смена стиля | канцелярит → разговорный → формальный в одном тексте |

### Структурные и стилевые (#16–21)
| # | Паттерн | Что делать |
|---|---------|------------|
| 16 | Избыток длинного тире | Оставить 1–2 на абзац |
| 17 | Избыток жирного шрифта | Жирным — только ключевые термины |
| 18 | Эмодзи-списки | Убрать, если не соцсети |
| 19 | Избыточные таблицы | Таблица на 2–3 строки → список |
| 20 | Следы Markdown в тексте | Убрать `**жирный**`, `#заголовки` |
| 21 | Нарушение иерархии заголовков | H1 → H2 → H3, без прыжков |
