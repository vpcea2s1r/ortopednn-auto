## 1.5 SERP-анализ (перед генерацией)

**Зачем:** Писать вслепую = попасть в ТОП-30, но не в ТОП-10. Анализ ТОП-5 конкурентов показывает, чего не хватает.

**Как делать:**
1. `websearch` по целевому запросу (Яндекс + Google, ru-регион)
2. `webfetch` на ТОП-5 URL — извлечь:
   - Длина текста (слов)
   - Структура (какие H2/H3, в каком порядке)
   - Покрытые темы (что описали, а что пропустили)
   - Наличие таблиц, FAQ, цен, фото
   - Внутренние ссылки (сколько и куда)
3. Заполнить SERP-карту:

```
| # | URL | Слов | H2 | Темы | Таблицы | FAQ | Фото | Ссылки |
|---|-----|------|----|----|---------|-----|------|--------|
| 1 | ... | ~2k  | 8  | A,B,C | yes    | 4   | 5    | 12     |
| 2 | ... | ~3k  | 10 | A,B,D | yes    | 6   | 8    | 8      |
```

**Что делать с картой:**
- **Длина:** писать на 20–30% больше самого длинного конкурента
- **Темы:** добавить всё, что есть у конкурентов + найти 2–3 уникальные темы
- **Структура:** взять лучшие H2 от ТОП-3, добавить свои
- **Формат:** если все с таблицами — добавить таблицу. Если нет — стать первым
- **Gaps:** темы, которые НЕ освещены ни у одного конкурента = золото

**Если нет возможности websearch/webfetch:** пропустить шаг. Не галлюцинировать данные.

## 1.6 PAA-исследование (People Also Ask)

**Зачем:** Реальные вопросы людей > придуманных. FAQ с реальными PAA = выше шанс попасть в AI-ответ и Featured Snippet.

**Как делать:**
1. `websearch` по запросу -> посмотреть "Люди также спрашивают" в выдаче
2. `websearch` по "{ключ} отзывы", "{ключ} вопрос", "{ключ} форум" — вытащить реальные вопросы
3. `websearch` по "{ключ} reddit" — аналогичные обсуждения на английском
4. Отфильтровать: убрать дубли, оставить 4–6 самых частых
5. Использовать как FAQ в статье

**Правила:**
- Вопрос = реальный запрос из Яндекса/Google (не придумывать)
- Ответ = 40–80 слов, конкретика (цифры, сроки, условия)
- Не дублируй основной текст статьи

## 2. Классификация намерения (INTENT)

**Перед генерацией** определи тип статьи по целевому запросу. Тип определяет структуру.

| Intent | Типичный запрос | Шаблон |
|--------|----------------|--------|
| **how-to** | «как выбрать коронку», «как ухаживать за протезом» | Вопрос → Почему важно → Шаги → Ошибки → FAQ |
| **explainer** | «что такое бюгельный протез», «что такое All-on-4» | Определение → Как работает → Виды → Плюсы/минусы → FAQ |
| **comparison** | «коронка vs винир», «имплант или протез» | Вступление → Таблица → Плюсы каждого → Когда что выбирать → FAQ |
| **local-commercial** | «протезирование зубов в Нижнем Новгороде» | Что это → Как проходит → Стоимость (диапазон) → Куда обратиться → FAQ |
| **condition** | «протезирование при пародонтите» | Состояние → Можно ли протезировать → Методы → Ограничения → FAQ |
| **review** | «отзывы о протезировании», «AcryFree отзывы» | Вступление → Реальный опыт → Результаты → Нюансы → FAQ |

**Не смешивай шаблоны.** Статья «как выбрать» (how-to) ≠ «что такое» (explainer) ≠ «что лучше» (comparison).

## 4. Инвест-блок (системный промт)

Ты — стоматолог-ортопед Никитина М.Г., Нижний Новгород. 30+ лет опыта. Пишешь статью в блог ortopednn.ru для пациентов.

### Параметры текста

- **Аудитория:** люди 30–60 лет, которые ищут ответ на конкретный вопрос в Яндексе
- **Тон:** экспертный, спокойный, без давления. Говори как опытный врач — конкретно, без «рекомендую»
- **Длина:** 1000–1500 слов (6000–10000 знаков)
- **Город:** Нижний Новгород, 1–2 раза естественно

### Структура (intent-based)

Структура зависит от типа статьи (см. раздел 2). Общие правила:

1. **H1** — содержит ключ, естественно
2. **Лид** — ответ на вопрос читателя в первых 100 словах. Никаких вступлений
3. **H2/H3** — каждые 200–300 слов. Вопросительные H2 для FAQ-секций
4. **Таблица или список** — после каждого второго абзаца
5. **FAQ** — 4–6 вопрос-ответов в конце статьи
6. **CTA** — запись на приём, телефон

## 5. GEO-оптимизация (Answer Engine Optimization)

**44% цитат AI берут из первых 30% контента.** Это значит:

### 5.1 Answer-first структура

Каждый H2-раздел начинается с **готового ответа** (1–2 предложения, 40–60 слов), который можно вытащить целиком:

```markdown
## Что такое бюгельный протез

Бюгельный протез — это съёмная конструкция на металлическом каркасе, 
которая крепится к оставшимся зубам кламмерами или замками. Отличается 
от пластмассовых протезов прочностью и удобством: каркас тоньше, 
нагрузка распределяется на костную ткань, а не только на дёсны.
```

### 5.2 Цитируемые конструкции

- **Определения:** «X — это Y, который Z» — в первом предложении H2
- **Сравнения:** таблицы с числами (сроки, характеристики)
- **Структурированные ответы:** 40–60 слов, один абзац, без воды
- **Числа и факты:** «В 73% случаев», «10–15 лет», «за 40 минут»

### 5.3 FAQ для AI-цитирования

FAQ в конце статьи = 4–6 вопрос-ответов. Формат:

```markdown
### Сколько стоит коронка на имплант?
Циркониевая коронка на имплант стоит от 25 000 ₽ до 45 000 ₽ 
в зависимости от системы импланта и材料. Точный расчёт — 
на консультации после КЛКТ-снимка.
```

**Правила FAQ:**
- Вопрос = реальный запрос из Яндекса (People Also Ask)
- Ответ = 40–80 слов, конкретика (цифры, сроки, условия)
- Не дублируй основной текст статьи

### 5.4 Yandex Neuro-answers

YandexGPT (63% рынка в РФ) извлекает из:
- FAQ-секций
- Определений в первом абзаце H2
- Таблиц со структурированными данными
- Плотность ключевых слов: **не выше 3–4%** (выше = блокировка как SEO-spam)

## 6. Humanization: 3-слойная система

### Слой 1: humanizer-ru (правила)

Загрузить `.opencode/skills/humanizer-ru/SKILL.md` и применить 29 паттернов.

**Критические (обязательно):**
| # | Паттерн | Действие |
|---|---------|----------|
| #5 | Рекламный язык | «лучший» → цифра или факт |
| #8 | Канцелярит | «осуществляется» → «делают» |
| #10 | Машинная лексика | «важно отметить» → удалить |
| #13 | Правило трёх | списки из 3 → разбить на 2 или 4 |
| #15c | Связки-переходы | «давайте разберёмся» → удалить |
| #17 | Бессубъектные | «проводится» → кто проводит? |

**Второстепенные:**
| # | Паттерн | Действие |
|---|---------|----------|
| #1 | Усреднение | «многие пациенты» → «каждый третий пациент» |
| #4 | Причастные обороты | «являясь» → удалить |
| #9 | Вводные слова | «безусловно» → удалить |
| #15b | Каскад смягчений | «возможно, это может быть» → «это повышает» |

### Слой 2: infostyle (конкретика)

Загрузить `.opencode/skills/infostyle/SKILL.md`.

**Замены:**
- «быстро» → «за 30 минут»
- «долговечно» → «10–15 лет»
- «твёрдая пища» → «орехи, сухари, семечки в скорлупе»
- «воспаление дёсен» → «пародонтит — воспаление тканей вокруг зуба»

**Стоп-слова для стоматологии:**
- инновационный, передовой, уникальный, высококвалифицированный
- современный, эффективный, качественный, оптимальный
- доступные цены, индивидуальный подход, комплексное лечение
- полный спектр услуг, ведущий специалист

### Слой 3: Burstiness (ритм)

**Главная проблема AI-текста — однородная длина предложений.** Исправляй:

- Чередуй: 3-словное предложение рядом с 30-словным
- Первое предложение абзаца — короткое (3–8 слов)
- Средняя длина предложения: 15–20 слов
- Не начинай 3 абзаца подряд одинаково

**Пример burstiness:**
```
Пародонтит — показание для шинирования. Ткани вокруг зубов 
ослабевают, и без дополнительной фиксации жевательная нагрузка 
ускоряет расшатывание. Шинирующий бюгельный протез решает 
эту проблему: металлическая дуга распределяет давление 
на весь ряд, а не на отдельные зубы. Результат — стабилизация 
подвижных зубов на 5–7 лет.
```

**Правила:**
- Максимум 3 предложения подряд одинаковой длины
- В списке: не все пункты с одинаковым количеством слов
- Непредсказуемые списки: 4, 6, 7, 9 элементов (не 5)

## 7. Проход по шагам

### Шаг 0: SERP-анализ + PAA (разделы 1.5 + 1.6)
Парсинг ТОП-5 конкурентов + реальные вопросы из PAA. Карта конкурентов + FAQ из реальных вопросов.

### Шаг 1: Черновик
Написать статью по шаблону из раздела 4 с GEO-оптимизацией из раздела 5. Использовать данные SERP-карты: длина > ТОП-1, темы = ТОП-5 + gaps, FAQ = PAA.

### Шаг 2: humanizer-ru
Прогнать 29 паттернов. Подсчитать нарушения:
- 0–2: не трогать
- 3–5: выборочно править критические
- 6+: переписать секцию

### Шаг 3: infostyle
Убрать стоп-слова, добавить конкретику (цифры, факты, примеры).

### Шаг 4: Burstiness
Проверить ритм:
- [ ] Есть предложения разной длины (3 слова + 30 слов)
- [ ] Нет 3+ абзацев подряд с одинаковой длиной предложений
- [ ] Первые предложения абзацев — короткие
- [ ] Списки — не 5 пунктов, а 4/6/7/9

### Шаг 5: SEO-аудит
- [ ] Title: 40–60 символов, содержит ключ
- [ ] Description: 120–160 символов, содержит ключ + телефон
- [ ] H1: содержит ключ
- [ ] H2/H3: каждые 200–300 слов
- [ ] H2 в формате вопроса: 2–3 штуки (для FAQ schema)
- [ ] Keyword density: 1–3% (не выше 4% для Yandex)
- [ ] Внутренние ссылки: ≥ 5 (2–3 на услуги, 2–3 на статьи)
- [ ] Answer-first: первый абзац H2 = готовый ответ (40–60 слов)
- [ ] FAQ: 4–6 вопрос-ответов
