# Writer.md — Пайплайн написания статей для ortopednn.ru

## Build Stability Rule
- Любое изменение кода, JSON-схем или пайплайна — проверять совместимость с существующими данными
- Перед push — `npm run build` без ошибок
- При изменении полей в JSON — обновить все существующие файлы или добавить fallback на старый формат

## Проверка уникальности (перед генерацией)

Перед рерайтом — проверить, нет ли уже статьи на эту тему:

1. Поиск в `src/pages/blog/index.astro` — совпадение title (>=70%)
2. Поиск в `data/drafts/*.meta.json` — совпадение title
3. Если есть дубликат — остановиться, сообщить пользователю
4. Если нет — продолжать генерацию

### Категоризация статей
Новая статья должна быть отнесена к одному из кластеров. Категория указывается в `data/blog-articles.ts`:
- `koronki` — Коронки (всё о коронках, винирах, вкладках, материалах)
- `semnye-protezy` — Съёмные протезы (акрил, нейлон, AcryFree, уход, ремонт)
- `byugelnye-protezy` — Бюгельные протезы (замки, кламмеры, уход)
- `mosty` — Мостовидные протезы (мосты, сравнение с коронками)
- `implanty` — Протезирование на имплантах (All-on-4, альтернативы)

Если статья не относится ни к одному кластеру — оставить поле `category` пустым.

### Автоматическая проверка (бот)
Бот проверяет:
- Существующий slug в `data/blog-articles.ts`
- Существующий slug в drafts
- Совпадение title (>60% символов совпадают)

### Ручная проверка (статья написана вручную)
- Открыть `data/blog-articles.ts`, проверить список articles
- Открыть `data/drafts/`, проверить *.meta.json

## Промт для AI (генерация черновика)

### Инвест-блок (вставь в промт перед генерацией)

Ты — стоматолог-ортопед Никитина М.Г., Нижний Новгород. Пишешь статью в блог ortopednn.ru для пациентов.

Параметры текста:
- **Аудитория:** люди 30–60 лет, которые ищут ответ на конкретный вопрос в Яндексе
- **Тон:** спокойный, экспертный, без давления. Не «рекомендую», а «используйте». Не «расскажу», а сразу факты.
- **Длина:** 1000–1500 слов
- **Цель:** ответить на вопрос читателя, показать экспертизу, подвести к записи (тел.: +7 (831) 233-33-44)
- **Город:** Нижний Новгород (упомянуть 1–2 раза естественно)
- **Источники:** PubMed, Cochrane, зарубежные стоматологические сайты (dentaly.org, animated-Teeth.com, dentistry.com) — информация используется как фон, ссылки и названия источников НЕ указываются
- **Дата:** сегодняшняя

Анкета статьи:
- Тема: {тема}
- Целевой запрос: {ключевая фраза}
- Дополнительные ключи: {LSI-фразы}
- Связанные услуги: {ссылки на /services/}

### Структура статьи

1. **Заголовок H1** — содержит ключ, естественно
2. **Лид (1 абзац)** — главный ответ сразу. Никаких вступлений.
3. **Постановка вопроса (H2)** — почему это важно, кому актуально
4. **Основной ответ (H2 + H3)** — факты, цифры, сроки, механизмы
5. **Таблица или список** — визуальное резюме
6. **Когда к врачу (H2)** — чёткие признаки
7. **CTA** — «Записаться: +7 (831) 233-33-44» (естественно)

### Чеклист «до генерации» (проверь промт)

- [ ] В промте указан город (Нижний Новгород)
- [ ] В промте указан телефон (+7 (831) 233-33-44)
- [ ] В промте запрещены цены
- [ ] Информация из PubMed/зарубежных источников подаётся как факты, без ссылок и названий источников
- [ ] Нет общих фраз («инновации», «передовые технологии», «высококвалифицированный»)

## Проход 1: humanizer-ru (убрать AI-следы)

### 29 паттернов AI-текста (сокращённый чеклист)

Критические (убрать обязательно):
- `#5` Рекламный язык: «лучший», «топовый», «эффективный» без цифр
- `#8` Канцелярит: «осуществляется», «представляет собой», «является», «профессиональный/ая»
- `#10` Машинная лексика: «рекомендую», «необходимо», «следует», «важно отметить»
- `#13` Правило трёх: списки из трёх пунктов с одинаковой структурой (разбивать или сокращать)
- `#15c` Связки-переходы: «расскажу, как…», «давайте разберёмся…», «в этой статье мы…»
- `#17` Бессубъектные: «проводится», «выполняется» → кто проводит?

Второстепенные (проверить):
- `#1` Общие слова без конкретики
- `#7` Тавтология в соседних предложениях
- `#9` Вводные слова-паразиты («безусловно», «несомненно», «очевидно»)
- `#16` Искусственные вопросы («Знаете ли вы?», «Задумывались ли вы?»)

### Дерево решений

| Признаков | Действие |
|-----------|----------|
| 0–2 | Не трогать |
| 3–5 | Выборочная правка найденного |
| 6+ | Переписать секцию или всю статью |

### Техника прохода

1. Прочитать вслух — найти неестественные места
2. Заменить «я рекомендую/советую» на факты
3. Убрать «необходимо», «следует», «важно» — заменить на конкретные инструкции
4. Разбить/сократить тройные списки
5. Убрать риторические вопросы
6. Заменить безличные конструкции на личные («пациент получает» → «вы получите»)

## Проход 2: infostyle (адаптация под пациентов)

### Убрать штампы

Из стоп-листа infostyle:
- инновационный, передовой, уникальный, высококвалифицированный, современный, эффективный, качественный
- доступные цены, индивидуальный подход, комплексное лечение
- на рынке с 20.. года, полный спектр услуг

### Добавить конкретику

- Каждое утверждение → цифра или срок
  - «быстро» → «за 30 минут»; «долго» → «10–15 лет»
  - «дорого» → «от 25 000 ₽» (нет — цены убраны) → «сравните с ценой повторного протезирования»
- Абстракции → примеры
  - «твёрдая пища» → «орехи, сухари, леденцы, семечки в скорлупе»

### Разбить на сканы

- Абзацы по 2–4 предложения
- Одна мысль → один абзац
- Подзаголовки H2/H3 каждые 200–300 слов
- Таблицы и списки — после каждого второго абзаца

### Язык

- Заменить англицизмы:
  - «имплантация» → допустимо (устоявшийся термин)
  - «пародонтит» → «воспаление тканей вокруг зуба» (для пациентов)
  - «композит» → «пломбировочный материал»
- Минимум пассивного залога
- Прямой порядок слов в предложениях

## Финальный чеклист

- [ ] URL: `/blog/{slug}/` (латиница, без стоп-слов)
- [ ] Sitemap: добавится автоматически через `@astrojs/sitemap`
- [ ] Каноническая ссылка

### Контент
- [ ] Нет цен (нигде — ни в тексте, ни в подписях, ни в ссылках)
- [ ] Нет AI-следов после humanizer-ru

- [ ] Есть ссылки на услуги (2–3 шт., естественные)
- [ ] Есть телефон в конце или CTA
- [ ] Город упомянут 1–2 раза
- [ ] Факты из источников подаются без ссылок (как общеизвестные данные)

### Категория и ссылки
- [ ] `category` указана в `data/blog-articles.ts`
- [ ] Статья попала в правильный кластер (проверить `/blog/{cat}/`)
