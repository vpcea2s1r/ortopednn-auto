# Контент-машина ortopednn.ru

## Problem Statement
Как автоматически собирать поисковые запросы и темы конкурентов по ортопедической стоматологии, генерировать под них статьи и выводить ortopednn.ru в топ Яндекса по Нижнему Новгороду.

## Цель
Топ-1 Яндекса по Нижнему Новгороду (Google второстепенно).
Целевая аудитория: Одноклассники + Яндекс.Дзен ("там все без зубов трутся").

## Решение: Партизанский контент-фабрика (модульная архитектура)

Три модуля:
1. **Скрапер** — собирает темы с сайтов конкурентов
2. **Генератор** — AI-генерация статей (Pollinations API)
3. **Публикатор** — публикация на ortopednn.ru + Дзен + ОК

## Архитектура

```
Скрапер → Банк тем → Генератор (AI) → Черновик → Telegram бот → Одобрение → Публикатор
                    (data/scraped-topics.json)         (draft .astro)   /  \      ├→ ortopednn.ru
                                                                   Approve Reject  ├→ Дзен (потом)
                                                                                    └→ ОК (потом)
```

## Telegram Bot Approval Flow (выбрано)

Бот (@ortopednn_bot, Python, GitHub Actions poll) присылает черновик статьи пользователю:
1. `/topics` — сколько тем собрано, список
2. Генерация по команде — `/generate [slug]` — AI пишет черновик
3. Бот присылает текст статьи в Telegram
4. Кнопки: ✅ Одобрить / ❌ Отклонить / ✏ Править
5. При одобрении — публикация на сайт

## Дополнительные источники семантики

### 1. Yandex Webmaster API

**Что собираем:** поисковые запросы, по которым конкуренты находят пользователей. Яндекс.Вебмастер API (`webmaster:hostinfo`) позволяет:
- Получить топ-500 поисковых запросов для своего сайта (`GET /v4/user/{userId}/hosts/{hostId}/search-queries/top-queries`)
- Получить динамику показов/кликов/CTR по запросам
- Статистику по страницам в выдаче

**Ограничения:** API даёт данные только по СВОИМ сайтам. Для конкурентов нужен обходной путь:
- **Вариант A:** Яндекс.Wordstat через парсинг (неофициально, риск блокировки)
- **Вариант B:** Keys.so / Serpstat / SEMrush API (платно)
- **Вариант C:** Ручной сбор — заводим конкурентов в Яндекс.Вебмастер? Невозможно.

**Решение для v1:** Используем Yandex API для своего сайта (уже есть OAuth токен с правами `webmaster:hostinfo` + `webmaster:verify`). Собираем:
1. Свои текущие поисковые запросы — на какие запросы уже показываемся
2. Нулевые запросы — что ищут, а мы не показываемся (gap-анализ)
3. Динамику: какие запросы растут/падают

```
{Yandex API} → {Свои поисковые запросы} → {gap-анализ с scraped-topics.json} → {Темы для контент-плана}
```

### 2. Ручное добавление сайта + AI-краулер

**Сценарий:** Пользователь (или AI-агент) видит сайт конкурента с интересной страницей → добавляет URL → AI анализирует и пополняет банк тем.

**Флоу:**
```
URL → AI-краулер → Анализ страницы → Извлечение: H1, H2, FAQ, список услуг, связи → data/scraped-topics.json
```

**AI-краулер делает:**
1. Загружает HTML страницы
2. Извлекает: заголовок (H1), подзаголовки (H2-H3), вопросы из FAQ, списки услуг/материалов/цен
3. Классифицирует: статья / услуга / FAQ / сравнение
4. Определяет ключевую тему (о чём страница)
5. Сохраняет в `data/scraped-topics.json`

**Формат записи:**
```json
{
  "source": "https://drdentnn.ru/services/koronki",
  "competitor": "drdentnn.ru",
  "h1": "Коронки на зубы",
  "h2s": ["Какие бывают коронки", "Металлокерамика", "Цирконий"],
  "faq": ["Сколько служит?", "Больно ли ставить?"],
  "type": "service",
  "topic": "коронки на зубы виды и цены",
  "added": "2026-05-23",
  "collectedBy": "manual"
}
```

**Добавление:** через Telegram-команду `/analyze [url]` или скрипт `node scripts/scraper.js --url https://...`

### Telegram-бот интеграция (дополнительные команды)

| Команда | Описание |
|---------|----------|
| `/analyze [url]` | Добавить URL конкурента → AI анализирует → пополняет банк тем |
| `/topics` | Список собранных тем |
| `/gap` | gap-анализ: какие темы у конкурентов есть, а у нас нет |
| `/yandex-queries` | Запросы из Яндекс API |
