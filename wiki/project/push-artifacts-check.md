# Push Artifacts Check

Правило верификации перед и после push в GitHub через API.

## Причина

2026-06-02: при push 5 новых статей через GitHub API возникла проблема — stale entries в дереве коммитов. Файлы не существовали на диске (созданы task agent'ом в sandbox), но `base_tree` содержал ссылки на них от предыдущей попытки. Это привело к:
1. Cломанному CI build (blob content не совпадал)
2. Stale tree entries, которые не удалялись при повторном push
3. Необходимости сброса ветки и пересоздания чистого дерева

## Правило

### Перед push
1. **Проверить, что файлы существуют на диске** — каждый файл должен быть подтверждён `Test-Path` или `Get-Item`
2. **Проверить размер файлов** — если файл пустой или подозрительно маленький (< 100 bytes) — encoding сломан
3. **Запустить `npm run build`** — build должен проходить без ошибок
4. **Проверить `git status`** (если есть git-репозиторий) — убедиться, что не затрётся живой код

### При push через GitHub API
1. **Получить текущий `base_tree` SHA** из последнего коммита
2. **Собрать blobs ТОЛЬКО для файлов, которые реально существуют на диске** — каждый blob обязан быть создан из файла
3. **Проверить новые tree items** — в массиве `tree` не должно быть entry с одинаковыми `path` и разными `sha`
4. **Прочитать новый tree после создания** — убедиться, что все entries корректны (можно сделать `GET /git/trees/:sha`)
5. **Проверить, что старые stale entries не остались:** количество файлов в новом tree должно соответствовать ожидаемому

### После push
1. **Проверить CI build** — подождать завершения GitHub Actions
2. **Если CI упал** — не делать повторный push в ту же ветку без проверки состояния tree
3. **Проверить live-сайт** — открыть 2-3 ключевые страницы после деплоя

### Аварийное восстановление
Если push создал "грязное" дерево (stale entries, битые blobs):
1. Получить SHA коммита ДО проблемы (`git log` на локальной копии или через API)
2. Создать новое дерево от этого коммита с ТОЛЬКО корректными файлами
3. Создать commit + update ref
4. Сбросить ветку force-push'ом через API (`sha = хороший_коммит, force = true`)
