# Humanizer-ru Integration

## Problem
Articles generated by the AI pipeline were detectable as AI-written by search engines:
- Identical structure (lead → table → h2 → list → FAQ) in every article
- No personal voice, no clinical examples from the doctor
- 29+ AI patterns in Russian text (связки-переходы, канцелярит, puffery)
- checkAiTells() only checked English patterns (delve, leverage, etc.) — useless for Russian

## Solution

### humanize() function
Created in server/agent-pipeline.js:459 — regex-based post-processor that:

| Pattern | Action |
|---------|--------|
| Связки-переходы | «стоит отметить», «в заключение хочется», «подводя итог» → remove |
| Канцелярит | «является», «представляет собой», «выступает в качестве» → simplify |
| AI-слова | «инновационный», «уникальный», «безусловно», «революционный» → remove |
| Размытые атрибуции | «эксперты считают», «исследователи отмечают» → remove |
| Каскад смягчений | «возможно, в некоторых случаях» → remove |
| Правило трёх | em-dash overuse capped at 1.5x paragraph count |
| Пустые фразы | «играет важную роль», «крайне важно» → remove |

### checkAiTells() enhanced
Added 3 new arrays for Russian patterns:
- RU_AI_PHRASES — 25+ banned phrases
- RU_AI_WORDS — 20+ banned words  
- RU_AI_PATTERNS — 6 regex patterns (не только...но и, деепричастия, меризм)
