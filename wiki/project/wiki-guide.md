# Как управлять wiki — руководство

## Команды для пользователя

Ты говоришь агенту одну из фраз — он выполняет нужную операцию.

### Ingest — добавить новый источник

```
запиши в wiki этот сайт/статью
добавь [ссылка] в базу знаний
прочитай и сохрани [источник] в вики
```

Агент:
1. Читает источник (webfetch)
2. Извлекает ключевые факты
3. Создаёт/обновляет страницы в `medical/` или `project/`
4. Перелинковывает связанные страницы
5. Обновляет index.md и log.md

### Query — спросить у wiki

```
что wiki знает про [тема]?
какой материал лучше для коронки? (по wiki)
найди в вики противоречия по [теме]
```

Агент:
1. Читает index.md → находит релевантные страницы
2. Читает найденные страницы
3. Отвечает со ссылками на wiki-страницы
4. Если ответ содержит новые ценные данные — сохраняет их в wiki

### Lint — проверить здоровье wiki

```
проверь wiki на целостность
сделай lint вики
найди битые ссылки в wiki
```

Агент:
1. Проверяет противоречия между страницами
2. Ищет устаревшие данные
3. Находит orphans (страницы без входящих ссылок)
4. Проверяет битые внутренние ссылки
5. Обновляет index.md
6. Пишет отчёт в log.md

### После сессии

```
зафиксируй итоги сессии в wiki
сохрани ключевые выводы в wiki
```

Агент записывает в log.md и создаёт/обновляет страницы с инсайтами.

## Когда wiki обновляется автоматически

- **После чтения нового источника** в рамках работы — агент сам предлагает сохранить в wiki
- **После ответа на сложный вопрос** — если ответ содержит синтезированные знания из нескольких источников
- **После написания статьи** — ключевые факты из статьи попадают в wiki

## Структура страницы (пример)

```markdown
# Тема

## Описание
Краткое сущностное описание.

## Ключевые факты
- Факт 1
- Факт 2

## Плюсы/Минусы (если применимо)

## Источники
- [Название](raw/source.md) или [внешняя ссылка](https://...)

## Связанные страницы
- [Связанная тема](related.md)
```

## Правила для агента

1. **Raw-источники** (`raw/`) — не редактировать, только читать
2. **index.md** — всегда актуален (обновлять при любом изменении)
3. **log.md** — append-only (только добавлять, не редактировать старые записи)
4. **Внутренние ссылки** — относительные, `[текст](page.md)`
5. **Внешние ссылки** — полные URL
6. **Битые ссылки** — не создавать. Если удаляешь страницу — убери все ссылки на неё

## Примеры

**Добавить статью PubMed:**
> «запиши эту статью в wiki» → (ссылка на PubMed)

**Спросить:**  
> «что wiki знает о бюгельных протезах на замках?»

**Синтез:**  
> «сравни по wiki импланты и мосты — что лучше при потере одного зуба?»

**Проверка:**  
> «сделай lint вики»
